4. A single decision tree may be a better choice than a decision forest (e.g., a random forest) in situations where:
1. Interpretability and Transparency Are Required
Context:
In fields like healthcare, law, or finance, decisions must be easily explainable to users, regulators, or stakeholders.

Why a tree is better:
A single decision tree provides a clear and visual path for how decisions are made — step-by-step — making it easy to understand and trust.

Contrast:
A decision forest is an ensemble of many trees and behaves like a "black box," making it difficult to explain individual predictions.

2. Low Resource Environments or Need for Fast Predictions
Context:
In real-time systems, mobile apps, or embedded devices where computational resources are limited (e.g., limited memory, processing speed).

Why a tree is better:
A single decision tree is lightweight, fast to evaluate, and requires minimal storage.

Contrast:
A decision forest has many trees that need to be evaluated, which consumes more memory and slows down prediction time.

✅ Summary
Aspect	Decision Tree Advantage
Interpretability	Easy to understand and explain decisions
Efficiency	Fast prediction with low computational cost
In short: A decision tree is preferable when explainability and efficiency are more important than maximizing predictive accuracy.

5. Naïve Bayes (NB) Classifier and Its Assumptions
a. Two Simplifying Assumptions in Naïve Bayes
Conditional Independence Assumption
Explanation: Naïve Bayes assumes that given the class label (e.g., "spam" or "not spam"), the features (e.g., words in a message) are conditionally independent of each other.
Example: If the class is "spam", the presence of the word "free" is treated as independent from the word "money", even though in reality they may co-occur often in spam messages.
Feature Relevance Assumption
Explanation: It assumes that all features contribute independently and equally to the final classification. That is, no feature is irrelevant, and no interaction between features is required for decision-making.
Example: Every word contributes its own probability score to whether a message is spam, regardless of context or word order.

6. NEAT (NeuroEvolution of Augmenting Topologies) Algorithm
a. Application of NEAT in a Mobile Robot
Example Application: Autonomous navigation in a dynamic environment

NEAT can evolve both the topology and weights of a neural network that controls a mobile robot’s behavior, such as obstacle avoidance and path planning.

The robot is equipped with sensors (e.g., LIDAR, ultrasonic, infrared) to detect distances to nearby objects.

These sensor readings form the input to the neural network, which outputs motor control signals (e.g., move forward, turn left/right).

Using NEAT, multiple generations of neural networks are evolved based on their fitness — e.g., how far the robot travels without hitting obstacles or how efficiently it reaches a target.

Over time, NEAT selects the best-performing networks and gradually evolves more effective topologies that adapt to environmental complexity, even adding hidden layers or connections as needed.

b. A Time-Consuming Aspect of Training or Tuning NEAT
Challenge: Designing and tuning the fitness function

The fitness function is critical in evolutionary algorithms like NEAT, as it guides the selection of "better" networks.

For a mobile robot, designing a fitness function that balances multiple competing objectives—such as avoiding collisions, minimizing travel time, and following legal or safe paths—can be very time-consuming.

If the fitness function is poorly designed, the algorithm might converge on suboptimal or undesired behaviors (e.g., the robot learns to spin in place to avoid collisions instead of navigating).

Iteratively adjusting the fitness function, running simulation tests, and ensuring that it aligns with real-world performance requires significant domain knowledge, experimentation, and debugging time.

7. Self-Supervised Learning (SSL)
a. Pretext and Downstream Tasks in SSL
Self-supervised learning trains models using automatically generated labels from the input data itself, allowing the model to learn useful representations without requiring human-annotated labels.
Pretext Task:
A task designed to teach the model to understand patterns or structure in the data without needing manually labelled ground truth.

The goal is not the task itself, but to learn meaningful representations.
Examples:
In NLP: Predicting the next word (e.g., in GPT), or masked words (e.g., BERT).

In computer vision: Predicting the rotation of an image, or reconstructing missing patches.

Downstream Task:
A real-world task (like classification, regression, or segmentation) that uses the representations learned during the pretext task.
After pretraining on the pretext task, the model is fine-tuned (or evaluated) on the downstream task using a small labelled dataset.

Examples:
Sentiment classification in text.
Image classification using features learned via self-supervised contrastive learning.

b. Comparison with Other Paradigms
i. Self-Supervised Learning vs. Supervised Learning

Aspect	                       Self-Supervised Learning	                              Supervised Learning
Label Type	       Uses labels generated from the input data itself	        Requires manually annotated labels
Data Efficiency	       More data-efficient (can use large unlabelled datasets)	Needs large labelled datasets
Training Objective      Learns representations via pretext tasks	                Directly optimizes for the task label
Usage	Pretraining for general features	Direct task-specific training
Key Difference: Supervised learning relies on external supervision, whereas self-supervised learning creates supervision internally using pretext tasks.

ii. Self-Supervised Learning vs. Unsupervised Learning
Aspect	                      Self-Supervised Learning	                  Unsupervised Learning
Learning Goal	         Learn useful features via proxy tasks         	Discover structure or patterns (e.g., clusters)
Label Involvement 	 Uses pseudo-labels from data              	No labels or proxy tasks involved
Examples	BERT, SimCLR, MoCo	K-Means, PCA, Autoencoders (unsupervised)
Output Use	Often followed by fine-tuning for downstream tasks	Mostly used for data exploration or preprocessing

Key Difference: While both don't use human labels, SSL involves a specific predictive task, making it more goal-directed than unsupervised learning, which is often exploratory.

8. Replay Techniques in Continual Learning and the Stability-Plasticity Trade-off
Continual learning aims to enable models to learn from a stream of tasks or data over time without suffering from catastrophic forgetting—the tendency for a model to forget previously learned information when trained on new data.
Role of Replay Techniques
Replay techniques are among the most widely used strategies to combat catastrophic forgetting. They involve reintroducing past data (or approximations of it) during training on new tasks.

Two main types:
Experience Replay (Rehearsal):
Stores a subset of past training data in a memory buffer.
During new task training, samples from the buffer are interleaved with current task data.
Helps the model revisit old knowledge, maintaining performance on previous tasks.

Generative Replay:
Instead of storing actual data, a generative model (e.g., GAN or VAE) is trained to synthesize past data.
This avoids storage concerns and privacy issues but introduces challenges in generation quality.
Effect: Replay helps anchor the model’s knowledge, reducing the chance of overwriting past parameters during new learning phases.

Stability-Plasticity Trade-off
Replay techniques help balance the trade-off between:

Stability: The model's ability to retain previous knowledge.

Plasticity: The model's ability to adapt to new knowledge.

Replay’s Role:

By revisiting past experiences during training, replay promotes stability, ensuring the model doesn't forget.

However, if too much emphasis is placed on old data (e.g., over-replay), it may reduce plasticity, making it harder for the model to learn new tasks effectively.

Conclusion:
Replay techniques moderate the stability-plasticity trade-off, allowing the model to incrementally learn while preserving prior task performance. Tuning the replay ratio, buffer size, and sampling strategy is key to maintaining this balance.

9. AI Algorithm in Self-Driving Cars: Convolutional Neural Networks (CNNs)
Algorithm: Convolutional Neural Networks (CNNs)
CNNs are a type of deep learning algorithm particularly well-suited for processing visual data, such as camera images from self-driving cars.

Role in Autonomous Driving
CNNs are extensively used in the perception system of self-driving cars to interpret and understand the surrounding environment. Their main contributions include:

Object Detection:

CNN-based models (e.g., YOLO, Faster R-CNN) detect and classify objects such as vehicles, pedestrians, traffic lights, and road signs in real time.

This enables the car to recognize obstacles and make safe navigation decisions.

Lane Detection:

CNNs can identify lane markings on the road, even under challenging conditions like poor lighting or occlusions.

This information is crucial for lane keeping and path planning.

Semantic Segmentation:

CNNs classify each pixel in an image into categories (e.g., road, sidewalk, vegetation), providing a detailed map of the driving environment.

This supports scene understanding and decision making.

Impact on Autonomous Driving
CNNs allow self-driving cars to:

Perceive the world with high accuracy and speed,

React to dynamic changes in traffic,

Maintain safety through object recognition and obstacle avoidance,

Enable advanced features like automated lane changing and parking.

Conclusion:
CNNs are foundational to how self-driving cars "see" the world. By extracting high-level features from raw sensor data, they empower the vehicle to understand and navigate complex real-world environments safely.
